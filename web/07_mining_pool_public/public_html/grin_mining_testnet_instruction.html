Grin Testnet Mining

Solo Mining Setup with IPOLLO G1 Mini

📋 Overview

This guide walks you through setting up solo mining on Grin testnet using an IPOLLO G1 Mini ASIC miner. You'll validate the core mining mechanics before deploying to mainnet.

Time needed: ~15 minutes (if node is already synced)

Requirements: Testnet node running, grin-wallet binary, IPOLLO on same network

1

Verify Testnet Node

Ensure your testnet Grin node is running and fully synchronized:

cd /opt/grin/node/testnet-prune
SECRET=$(cat .api_secret)
curl -s -u "grin:$SECRET" \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"get_status","params":[],"id":1}' \
  http://127.0.0.1:13413/v2/owner | jq '.result.Ok | {height, network, sync_status}'
✓ Expected output shows network: "Testnet" and sync_status: "Synced"
If not synced, wait ~30 minutes for the node to catch up
2

Initialize Testnet Wallet

Create a new testnet wallet or recover from seed:

POOL_WALLET_DIR="/opt/grin/pool-test"
mkdir -p "$POOL_WALLET_DIR"
cd "$POOL_WALLET_DIR"

# Create new wallet
grin-wallet init --testnet
Or recover from seed: Replace init with init -r and paste your seed when prompted
✓ Wallet data directory created at $POOL_WALLET_DIR/wallet_data
✓ Save the seed somewhere safe!
3

Configure Wallet TOML

Edit grin-wallet.toml to enable API listening:

WALLET_TOML="/opt/grin/pool-test/grin-wallet.toml"
nano $WALLET_TOML

Find the [api] section and update:

[api]
listen_addr = "0.0.0.0"
api_listen_port = 13415
enable_foreign_api = true
enable_owner_api = true

[server]
check_node_api_https = false
node_api_secret_path = "/opt/grin/node/testnet-prune/.foreign_api_secret"
Key points:
  • listen_addr = "0.0.0.0" allows IPOLLO to connect
  • api_listen_port = 13415 is testnet Foreign API port
  • node_api_secret_path points to testnet node secret file
4

Start Wallet Listener

Start the wallet API listener on port 13415:

cd /opt/grin/pool-test
grin-wallet --testnet listen 2>&1 | tee wallet.log &

Verify it's listening:

sleep 2
ss -tlnp | grep 13415
✓ Should show: LISTEN 0 128 0.0.0.0:13415
5

Start Stratum Server

Start the Node.js stratum server:

cd /opt/grin/pool-test/back-end-pool
npm start

Expected output:

[2026-05-15T...] Stratum server listening on port 3333
[2026-05-15T...] Pool API listening on port 8080

Verify stratum is listening:

ss -tlnp | grep 3333
✓ Port 3333 should be in LISTEN state
6

Configure IPOLLO Web UI

Access IPOLLO web interface and set up mining pool:

  1. Open browser: http://<ipollo-ip>:8080
  2. Navigate to: Pool Settings → Single Pool Configuration
  3. Enter:
    • Pool Address: <your-vps-ip>:3333
    • Worker Name: (leave blank or use miner ID)
    • Username: grin1<your-wallet-address>
  4. Click Save & Reboot

Find your wallet address:

cd /opt/grin/pool-test
grin-wallet --testnet info | grep "Listening"
Example: If wallet info shows grin1abc123xyz, use that in IPOLLO username field
7

Monitor Mining Activity

Check stratum stats in real-time:

curl http://localhost:8080/api/stratum/stats

Expected output (once IPOLLO connects):

{
  "active_connections": 1,
  "active_miners": 1,
  "sessions": [
    {
      "grin_address": "grin1abc123xyz...",
      "worker_name": "default",
      "difficulty": 1,
      "shares": 42,
      "online_seconds": 3600
    }
  ]
}

Check pool stats:

curl http://localhost:8080/api/pool/stats

Watch wallet balance (updates when blocks mature):

watch -n 5 'grin-wallet --testnet info | grep -A 2 "Confirmed"'
✓ Shares should increase as IPOLLO submits work
✓ Balance increases ~30 minutes after block is found (testnet confirmation time)
8

Troubleshooting

❌ IPOLLO not connecting

Check firewall and stratum listening:

sudo ufw allow 3333/tcp
sudo ufw reload
ss -tlnp | grep 3333

Verify stratum is listening on 0.0.0.0:3333 (not just 127.0.0.1)

❌ Wallet not syncing

Check wallet sync status:

grin-wallet --testnet info

If lagging node, restore wallet (takes 5-10 min):

grin-wallet --testnet restore

❌ No blocks found after hours

Verify IPOLLO is actually mining:

curl http://localhost:8080/api/stratum/stats

Should show increasing shares counter. If stuck, restart IPOLLO.

❌ Stratum crashes on startup

Check logs for errors:

npm start

Common issue: Wallet listener not ready. Wait 5 seconds after starting wallet, then start stratum.

Service Testnet Port Mainnet Port Purpose
Node API 13413 3413 Health checks, block verification
Wallet Foreign API 13415 3415 Stratum connects here for payments
Wallet Owner API 13420 3420 Admin wallet operations
Stratum Server 3333 (or custom) IPOLLO connects here
Pool HTTP API 8080 (or custom) Monitoring & stats endpoints

❓ Frequently Asked Questions

Q: How long until I find a block?

Testnet IPOLLO G1 Mini typically finds a block every 1-3 hours depending on difficulty. Check /api/pool/stats to see estimated pool hashrate. Once found, it takes 6 testnet blocks (~30 min) to mature and credit your balance.

Q: Can I mine mainnet with this setup?

Yes! Use the same procedure but with mainnet ports (3413, 3415, 3420 instead of 13413, 13415, 13420) and omit the --testnet flag from grin-wallet commands. Mainnet blocks mature after 1,441 blocks (~5 days).

Q: Can I add more IPOLLO units?

Yes! Configure each IPOLLO with the same pool address <vps-ip>:3333 but different worker names or usernames. Stratum automatically handles multiple connections and tracks shares per miner.

Q: How do I restart if something breaks?

Kill existing processes and restart in order:

pkill -f "grin-wallet.*listen" && sleep 1
grin-wallet --testnet listen & sleep 2

pkill -f "node.*back-end-pool" && sleep 1
cd /opt/grin/pool-test/back-end-pool
npm start

Q: Can I run testnet and mainnet simultaneously?

Yes! Use separate directories for each network's wallet (e.g., /opt/grin/pool-test for testnet, /opt/grin/pool-main for mainnet). Stratum can listen on different ports (3333 for testnet, 3334 for mainnet) and serve both.
