The regional connect endpoints shown on the public dashboard — and the live health of each region's gateway. A card lights up green while miners are mining that region (or its WireGuard tunnel is freshly handshaking).

How regions work — add a gateway in 4 steps (Model C)

A region has two halves: the connect card you declare here (what miners see), and the gateway — a thin box running just HAProxy + WireGuard that forwards miners' stratum to this pool over a tunnel (no node, no wallet). The pool stays the single brain. Declare the card here, add the gateway's WireGuard peer on this box, then stand the gateway up.

  1. Declare the region here (form below): pick a short region key (e.g. nyc), its country, and the stratum host. Save → the card appears on the public dashboard as ○ No signal.
  2. Point DNS: make the stratum host (e.g. nyc.grinium.com) resolve to the gateway box's IP.
  3. On this pool box: run 07_grin_mining_public_pool.sh → W) Multi-region → Add a gateway peer with the same region key. It assigns a tunnel IP + internal region port and prints the values to hand to the gateway operator.
  4. On the gateway box: run 07_grin_mining_public_pool.sh → option 2) Regional gateway, Install → Configure (paste the values from step 3) → Bring up tunnel → Start. The card flips to ● Online once miners connect.

Single-box pool? You don't need gateways — the pool auto-registers its own region and miners just use that one host. Add regions here only when you're expanding to other locations.
This page ≠ "Public Stratum Host" in Pool Info — that field is only the cosmetic fallback host, not a region/gateway registration.

What goes in each field? (click for a quick guide)
Region key — STEP 1
Short permanent identifier for this location. Suggestion: a 3-letter airport (IATA) code — nyc, lax, ams, yyz, han — or a 2-letter country code (us, nl, vn) when a country has one region. Shares get tagged with it, so it can't be renamed later, and it MUST exactly match the key you give in STEP 3 when adding the gateway's WireGuard peer (Script 07 → W → Add a gateway peer).
Label / City
Human-friendly name shown on the public connect card (e.g. New York). Cosmetic — doesn't affect routing.
Country & country code
Group regions by country on the dashboard (two US regions appear under one heading). The ISO-2 code (US, NL) draws the flag emoji. Leave the code blank for no flag.
Stratum host — STEP 2
The hostname miners point their rigs at for this region (e.g. nyc.grinium.com). Create a DNS A-record for it that resolves to the gateway box's public IP. Enter the host only — the shared stratum port is appended automatically.
Active
When on, the card shows on the public connect grid (as “No signal” until the gateway is up — STEPS 3 & 4). Turn off to hide a region without deleting it.
