Regions

The regional connect endpoints shown on the public dashboard — and the live health of each region's satellite. A card lights up green only while a satellite for that region keeps reporting in.

How regions work — set one up in 4 steps

A region has two halves: the connect card you declare here (what miners see), and the satellite — a separate box running the node + stratum proxy that actually does the mining and reports back. Declare the card here first, then stand up the satellite with the matching region key.

  1. Declare the region here (form below): pick a short region key (e.g. nyc), its country, and the stratum host. Save → the card appears on the public dashboard as ○ No signal.
  2. Point DNS: make the stratum host (e.g. nyc.grinium.com) resolve to the satellite box's IP.
  3. On the satellite box: run 07_grin_mining_public_pool.sh → option 2) Satellite agent, entering the same region key, this pool's hub URL (https://<your-pool-domain>), and the shared secret.
  4. Enable ingestion on the hub: set hub_shared_secret (matching the satellite) in /opt/grin/conf/grin_pubpool.json and restart the pool. The card then flips to ● Online from the satellite's heartbeat.

Single-box pool? You don't need satellites — the pool auto-registers its own region and miners just use that one host. Add regions here only when you're expanding to other locations.
This page ≠ "Public Stratum Host" in Pool Info — that field is only the cosmetic fallback host, not a region/satellite registration.
