Regions

The regional connect endpoints shown on the public dashboard — and the live health of each region's gateway. A card lights up green while miners are mining that region (or its WireGuard tunnel is freshly handshaking).

How regions work — add a gateway in 4 steps (Model C)

A gateway is a small extra server in another location. It runs only HAProxy + WireGuard — no grin node, no wallet, no database. Miners in that location connect to the gateway, and it forwards their stratum traffic through an encrypted tunnel to this pool box. This box stays the single brain: all shares, blocks and payouts are still handled here.

Every region needs one region key — a short airport-style code like nyc, sgn, ams. You will type the exact same key in steps 1, 3 and 4. Then, one step at a time:

  1. STEP 1 — Declare the region (on this page).
    Fill in the form below: region key, city, country, and the stratum hostname miners will use (e.g. sgn.grinium.com). Click Save region.
    Result: the card appears on the public dashboard as ○ No signal.
  2. STEP 2 — Point DNS at the gateway box.
    At your DNS provider, create an A-record so the stratum hostname from step 1 resolves to the gateway box's public IP (not this pool's IP).
    Result: miners who connect to that hostname reach the gateway box.
  3. STEP 3 — Register the gateway on THIS pool box (SSH).
    Run 07_grin_mining_public_pool.sh → W) Multi-region → Add a gateway peer. Enter the same region key. The script assigns a tunnel IP + an internal region port and prints the connection values.
    Result: a printout of values to hand to whoever runs the gateway box.
  4. STEP 4 — Set up the gateway box itself (SSH).
    On the gateway server, run 07_grin_mining_public_pool.sh → 2) Regional gateway, then in order: 1) Install → 2) Configure (paste the values from step 3) → 3) Bring up tunnel → 4) Start.
    Result: the card on this page flips to ● Active once miners start mining through it.

Single-box pool? You don't need gateways — the pool auto-registers its own region and miners just use that one host. Add regions here only when you're expanding to other locations.
This page ≠ "Public Stratum Host" in Pool Info — that field is only the cosmetic fallback host, not a region/gateway registration.

The Port :NNNN column (named after your public miner port) is an active wire test: on every refresh the pool box TCP-dials the region's public stratum host:port — the exact connection a miner's rig makes; the badge repeats the exact port it dialed. ✓ open = the gateway's forwarder answered (with round-trip ms); ✗ unreachable = refused / timed out / DNS failed. This is the miner port only — it is not the internal Region port, the WireGuard port, or the grin node's stratum port. Mining activity is the passive signal — no port is dialed for it: recent shares (miners actually mining the region) plus, when available, the WireGuard tunnel's last handshake; Active <3 min, Stalling 3–10 min, Silent >10 min. Port ✓ with No signal simply means the gateway is up but no miner is on it yet. A hidden tag on a region name means you've unchecked "Visible on site" in the form below — that region is hidden from the public connect grid.

What goes in each field? (click for a quick guide)
Region key — STEP 1
Short permanent identifier for this location. Suggestion: a 3-letter airport (IATA) code — nyc, lax, ams, yyz, han — or a 2-letter country code (us, nl, vn) when a country has one region. Shares get tagged with it, so it can't be renamed later, and it MUST exactly match the key you give in STEP 3 when adding the gateway's WireGuard peer (Script 07 → W → Add a gateway peer).
Label / City
Human-friendly name shown on the public connect card (e.g. New York). Cosmetic — doesn't affect routing.
Country & country code
Group regions by country on the dashboard (two US regions appear under one heading). The ISO-2 code (US, NL) draws the flag emoji. Leave the code blank for no flag.
Stratum host — STEP 2
The hostname miners point their rigs at for this region (e.g. nyc.grinium.com). Create a DNS A-record for it that resolves to the gateway box's public IP. Enter the host only — the shared stratum port is appended automatically.
Visible on site
A show/hide switch, nothing more — when on, the card appears on the public connect grid (as “No signal” until the gateway is up — STEPS 3 & 4). Turn off to hide a region without deleting it. It does not check whether the gateway is running; that's the Port and Mining activity columns.

Saving this form is STEP 1 of 4 — it only declares the card. Then: STEP 2 point DNS → STEP 3 add the WireGuard peer on the pool box (Script 07 → W) → STEP 4 stand up the gateway box (Script 07 → 2). Full walkthrough in “How regions work” above.
